Description
The challenge implements a stream cipher using an 8-bit Linear Feedback Shift Register (LFSR). The ciphertext is generated by XORing the plaintext with values produced by the LFSR.
The author assumed that using an LFSR would provide sufficient security, but the implementation only uses an 8-bit internal state, making brute-force attacks practical.

from Crypto.Util.number import bytes_to_long, long_to_bytes
from Crypto.Random import get_random_bytes
key = bytes_to_long(get_random_bytes(126))
def steplfsr(lfsr):
    b7 = (lfsr >> 7) & 1
    b5 = (lfsr >> 5) & 1
    b4 = (lfsr >> 4) & 1
    b3 = (lfsr >> 3) & 1
    feedback = b7 ^ b5 ^ b4 ^ b3
    lfsr = (feedback << 7) | (lfsr >> 1)
    return lfsr

def encrypt_lfsr(pt_bytes):
    output = bytearray()
    lfsr = key & 0xFF
    for p in pt_bytes:
        lfsr = steplfsr(lfsr)
        ks = lfsr
        output.append(p ^ ks)
    return bytes_to_long(bytes(output))
pt = b"[redacted]"
ct = encrypt_lfsr(pt)
print(long_to_bytes(ct).hex())

Vulnerability Analysis
The encryption only uses:
lfsr = key & 0xFF
This means that although a 126-byte random key is generated, only the lowest 8 bits are actually used.
Therefore, the number of possible LFSR states is:
2^8 = 256
A brute-force search across all possible seeds is trivial.
Since picoCTF flags begin with:
picoCTF{
we can test all 256 seeds and look for plaintext containing the string "pico".

def steplfsr(lfsr):
    b7 = (lfsr >> 7) & 1
    b5 = (lfsr >> 5) & 1
    b4 = (lfsr >> 4) & 1
    b3 = (lfsr >> 3) & 1

    feedback = b7 ^ b5 ^ b4 ^ b3
    lfsr = (feedback << 7) | (lfsr >> 1)
    return lfsr
def keystream(seed, n):
    s = seed
    out = []
    for _ in range(n):
        s = steplfsr(s)
        out.append(s)
    return out
c = "21c1b705764e4bfdafd01e0bfdbc38d5eadf92991cdd347064e37444e517d661cea9"
ct_bytes = bytes.fromhex(c)
for seed in range(256):
    ks = keystream(seed, len(ct_bytes))
    pt_guess = bytes(c ^ k for c, k in zip(ct_bytes, ks))
    if b"pico" in pt_guess:
        print(seed, pt_guess)

Key Takeaway
LFSRs are not secure when used directly as stream ciphers, especially with very small internal states.
The challenge generated a large random key but only used the lowest 8 bits, reducing the effective keyspace to 256 possibilities. This allowed a complete brute-force attack and recovery of the flag in seconds.
